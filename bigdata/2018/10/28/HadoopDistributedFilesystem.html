Introduction and The Design of HDFS

October-28-2018

Introduction

Distributed File Systems: Filesystems that manage the storage across a network of machines.

Hadoop comes in with a DFS called HDFS(Hadoop Distributed File System).

The Design of HDFS

HDFS is a filesystem designed for storing very large files with streaming data access patterns, running on clusters of commodity hardware

Streaming data access:

HDFS deals with processing large amount of data a large number of times over time hence it believes that the time taken to read the whole record is more important than the time taken to read one record (i.e average read rate is more important than instantaneous read rate).

Commodity Hardware:

Hadoop doesnâ€™t require reliable and expensive hardware. Instead it is designed to run on clusters of regular daily use computers. Even when a node(a computer) fails, hadoop is robust and can continue without any interruption.

Hence HDFS is not suitable for applications which require:

  • Low latency data access.
  • Working with a lot of small files.
  • Multiple writers and lot of random modifications to files.
